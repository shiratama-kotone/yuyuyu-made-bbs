エンドポイント一覧

メソッドパス認証説明
GET/api不要雑談の投稿・topic・規制状態を取得
POST/api不要雑談に投稿 / コマンド実行
GET/api/battle不要バトルスタジアムの投稿・topic・規制状態を取得
POST/api/battle不要バトルスタジアムに投稿 / コマンド実行
POST/account/register不要アカウント作成
POST/account/login不要ログイン（トークン発行）
GET/account/meBearer自分の情報を取得
GET/webhooksBearerWebhook一覧取得
POST/webhooksBearerWebhook登録
DELETE/webhooks/:idBearerWebhook削除
POST/webhooks/:id/testBearerWebhookテスト送信
GET/seedsearch?pass=不要シードID生成（マネージャー以上）
GET/health不要サーバー死活確認

投稿送信（POST /api）

JSONボディ

POST /api
Content-Type: application/json

{
  "name": "名前",
  "pass": "パスワード",
  "content": "内容"
}

IDの生成アルゴリズム

SHA-256(パスワード) → Base64 → 英数字のみ残す → 先頭7文字 → "@" を付与

例: パスワード hello → ID @LPJNulw

アカウント API

POST /account/register — アカウント作成

{
  "username": "my_name",
  "bbs_id":   "@AbCdEfG",
  "bbs_pass": "掲示板のパスワード"
}

レスポンス: { "token": "uuid..." }

POST /account/login — ログイン

{
  "bbs_id":   "@AbCdEfG",
  "bbs_pass": "掲示板のパスワード"
}

レスポンス: { "token": "uuid...", "username": "..." }

認証が必要なエンドポイント

ヘッダーに Authorization: Bearer <token> を付けてください。

Webhook

投稿が発生したときに登録したURLへPOSTリクエストが送信されます。

ペイロード

{
  "event_time": "2025-08-31T15:30:00.000Z",
  "id":         "@AbCdEfG",
  "name":       "投稿者名",
  "no":         42,
  "content":    "投稿内容",
  "channel":    "chat"
}

Webhook登録（POST /webhooks）

Authorization: Bearer <token>

{
  "url":     "https://example.com/webhook",
  "channel": "chat"
}

WebSocket

サーバーへのWebSocket接続URLは以下です。

wss://yuyuyu-made-bbs-server.onrender.com

受信メッセージ形式

// 新着投稿
{ "channel": "chat", "type": "post", "post": { ...postオブジェクト } }

// 全削除
{ "channel": "chat", "type": "clear" }

// 個別削除
{ "channel": "chat", "type": "delete", "nos": [3, 5] }

// destroy
{ "channel": "chat", "type": "destroy" }
