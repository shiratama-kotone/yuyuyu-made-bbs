データ取得 GET /api

レスポンス

{
  "topic": "今の話題（複数行も可）",
  "posts": [ ...投稿配列（新しい順）... ],
  "restriction": {
    "prevent":        false,   // 雑談の青ID投稿禁止中か
    "restrict":       false,   // banされた青IDの規制中か
    "stopActive":     false,   // /stop作用中か
    "stopUntil":      0,       // stop終了時刻（Unixミリ秒）
    "prohibitActive": false,   // /prohibit作用中か
    "prohibitUntil":  0        // prohibit終了時刻（Unixミリ秒）
  }
}

postsの各フィールド

rolenumber権限: 0=青ID 1=スピーカー 2=マネージャー 3=サミット 4=運営

addSuffixstring|nullIDの後ろのテキスト。未設定はnull

投稿送信 POST /api

リクエストボディ（JSON）

Content-Type: application/json

{
  "name":    "投稿者名",      // 必須 最大50文字
  "pass":    "パスワード",    // 必須 IDの生成に使用
  "content": "投稿内容"       // 必須 最大1000文字 コマンドも可
}

クエリパラメーターでも可

POST /api?name=名前&pass=パスワード&content=内容

レスポンス（成功）

{
  "message": "投稿成功",   // コマンド時はコマンド名（例: "/del"）
  "post": { ...投稿オブジェクト... }
}

エラーレスポンス

ステータスerror説明
400全フィールド必須name/pass/contentのいずれかが欠けている
429同じパスワードで1秒に1回まで連投制限
403権限不足 / 投稿禁止 / NGワード等各種制限
500データ処理失敗サーバーエラー

アカウントAPI

POST /account/register — アカウント作成

{
  "username": "my_name",    // 必須 英数字・_ 3〜20文字
  "bbs_id":   "@AbCdEfG",  // 必須 自分の掲示板ID
  "bbs_pass": "パスワード"  // 必須 IDと一致しないと登録不可
}

レスポンス: { "token": "uuid...", "message": "アカウントを作成しました" }

POST /account/login — ログイン

{
  "bbs_id":   "@AbCdEfG",
  "bbs_pass": "パスワード"
}

レスポンス: { "token": "uuid...", "username": "..." }

※ ログインするたびに新しいトークンが発行されます。古いトークンは無効になります。

GET /account/me — 自分の情報

ヘッダー: Authorization: Bearer <token>

{
  "username":   "my_name",
  "bbs_id":     "@AbCdEfG",
  "created_at": "2025-08-31T..."
}

Webhook API

投稿が発生したとき、登録したURLへPOSTリクエストが送信されます。1アカウント最大10件登録可能。

Webhookペイロード

{
  "event_time": "2025-08-31T15:30:00.000Z",  // 送信日時（ISO 8601）
  "id":         "@AbCdEfG",                  // 投稿者のハッシュID
  "name":       "投稿者名",
  "no":         42,                          // 投稿番号
  "content":    "投稿内容",
  "role":       0,                           // 権限レベル（0〜4）
  "channel":    "chat"                       // "chat" または "battle"
}

POST /webhooks — Webhook登録

Authorization: Bearer <token>
Content-Type: application/json

{
  "url":     "https://example.com/webhook",  // 必須
  "channel": "chat"                          // 任意 "chat"(default) or "battle"
}

レスポンス: { "webhook": { "id": 1, "url": "...", "channel": "chat", "created_at": "..." } }

DELETE /webhooks/:id — 削除

Authorization: Bearer <token>

POST /webhooks/:id/test — テスト送信

登録済みのWebhookへテスト用のペイロードを送信します。

WebSocket

wss://yuyuyu-made-bbs-server.onrender.com

接続後、そのまま待機するだけでリアルタイムに投稿イベントを受信できます。送信は不要です。

受信メッセージの形式

type説明追加フィールド
post新着投稿channel, post（投稿オブジェクト）
clear全削除（/clear実行）channel
delete個別削除（/del実行）channel, nos（番号配列）
destroy一括削除（/destroy実行）channel

受信例（JavaScript）

const ws = new WebSocket('wss://yuyuyu-made-bbs-server.onrender.com');

ws.onmessage = (e) => {
  const data = JSON.parse(e.data);
  if (data.channel !== 'chat') return; // チャンネルフィルタ

  if (data.type === 'post') {
    console.log('新着:', data.post.name, data.post.content);
  } else if (data.type === 'delete') {
    console.log('削除:', data.nos);
  }
};

シードID生成 GET /seedsearch

パスワードからランダムなIDペアを生成して返します。マネージャー以上のみ利用可能。

GET /seedsearch?pass=あなたのパスワード

レスポンス: { "seeds": [ { "pass": "a1b2c3d4", "id": "@AbCdEfG" }, ... ] }

生成数: マネージャー10件 / サミット100件 / 運営200件
